Network Cell Analyzer - Full Project Summary

Course: EECE 451   |   Date: March 2026

1 What Is This Application?

This is a two-part system (Android app + Python server) whose purpose is to:

  1. Collect real-time cellular network data from an Android phone (which operator you're connected to, signal strength, network type like 2G/3G/4G, etc.)
  2. Send that data to a server every 10 seconds
  3. Store it in a database and compute statistics (average signal, time spent on each operator, etc.)
  4. Display a web dashboard showing connected devices and recent measurements
Think of it as a network quality monitor — like a speedtest app, but focused on raw cellular signal metrics over time.

3 Every File Explained

SERVER Server-Side Files

FileWhat It Does
server/server.py The brain of the server. A Flask web app that: (1) receives cell data via POST /api/celldata, (2) stores it in SQLite, (3) computes statistics via GET /api/stats, (4) shows a web dashboard at / with connected devices and recent data. Auto-refreshes every 10 seconds.
server/requirements.txt Lists Python dependencies — just Flask==3.0.0
server/celldata.db The SQLite database file (auto-created). Stores all measurements with columns: operator, signal_power, sinr, network_type, frequency_band, cell_id, timestamp, device_id, ip_address, received_at

Server Database Schema

celldata table:
  id              INTEGER PRIMARY KEY AUTOINCREMENT
  operator        TEXT NOT NULL          (e.g., "Alfa", "Touch")
  signal_power    INTEGER NOT NULL       (dBm value)
  sinr            INTEGER                (Signal-to-Interference Ratio, nullable)
  network_type    TEXT NOT NULL          ("2G", "3G", "4G")
  frequency_band  INTEGER                (EARFCN/ARFCN/UARFCN, nullable)
  cell_id         TEXT NOT NULL          (Location Area Code - Cell ID)
  timestamp       TEXT NOT NULL          (ISO format from device)
  device_id       TEXT NOT NULL          (Unique device identifier)
  ip_address      TEXT                   (Client IP address)
  received_at     TEXT NOT NULL          (Server-side timestamp)

Server API Endpoints

EndpointMethodWhat It Does
/api/celldata POST Receives cell measurements from Android devices. Expects JSON with: operator, signalPower, sinr, networkType, frequencyBand, cellId, timestamp, deviceId. Stores in database. Returns {"status": "ok"}.
/api/stats GET Computes statistics from stored measurements. Supports query params: start, end (date range), device_id. Returns: connectivity per operator, connectivity per network type, avg signal per network type, avg signal per device, avg SINR per network type.
/ GET Web dashboard showing: number of active devices, device list with IPs and status, recent 20 measurements table. Auto-refreshes every 10 seconds. Devices marked "Active" if seen in last 30 seconds.

ANDROID Kotlin Source Files (6 files)

FileWhat It Does
MainActivity.kt The main screen. Has a text field for server URL, Start/Stop button, and displays live cell data (operator, signal, network type, etc.). Listens for broadcasts from the background service to update the UI in real-time.
CellInfoCollector.kt The data reader. Uses Android's TelephonyManager.getAllCellInfo() to read the phone's actual cellular connection. Handles 2G (GSM), 3G (WCDMA), and 4G (LTE) differently, extracting signal power, SINR, frequency band, and cell ID for each type.
CellInfoData.kt A simple data class — a container that holds one snapshot of cell info: operator, signalPower, sinr, networkType, frequencyBand, cellId, timestamp, deviceId. Like a struct.
CellInfoService.kt The background service. Runs as a foreground service (shows notification) so Android doesn't kill it. Every 10 seconds it: collects cell data → sends to server → broadcasts to UI. Keeps running even when the app is minimized.
ServerClient.kt The HTTP client. Uses OkHttp to send cell data to the Flask server (POST /api/celldata) and fetch statistics (GET /api/stats). Converts data to JSON using Gson.
StatisticsActivity.kt The statistics screen. User picks a date range, hits "Fetch", and it displays: % time per operator, % time per network type, average signal per network type, average signal per device, average SINR per network type.

9 What's NOT Done Yet (Remaining Steps)

1. Connecting Phone to Server Over a Network

Right now the app defaults to http://10.0.2.2:5000 which only works on the Android emulator (10.0.2.2 is a special alias for your PC's localhost from inside the emulator).

Option A: Same Wi-Fi Network (Easiest)
  1. Connect your phone and PC to the same Wi-Fi
  2. Find your PC's local IP (run ipconfig in Command Prompt, look for 192.168.x.x)
  3. Run the server with: python server.py (it already binds to 0.0.0.0)
  4. In the app, enter: http://192.168.x.x:5000
Option B: ngrok (Different Networks)

If your phone and PC are on different networks (e.g., phone on mobile data, PC on home Wi-Fi):

  1. Install ngrok: https://ngrok.com/download
  2. Run: ngrok http 5000
  3. ngrok gives you a public URL like https://abc123.ngrok.io
  4. In the app, enter that URL as the server address
  5. Note: ngrok provides HTTPS which works fine
Option C: Cloud Hosting

Deploy server.py to a cloud service (Heroku, Render, PythonAnywhere, etc.) so it has a permanent public URL.

2. Testing on a Real Device

  • The emulator returns dummy/empty cell info
  • You need to grant location + phone permissions when prompted
