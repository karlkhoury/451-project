================================================================================
  3. WHAT THE PROJECT IS — ONE-LINE AND ONE-PARAGRAPH SUMMARIES
================================================================================

ONE LINER:
"An Android app that monitors real-time cellular network quality (2G/3G/4G/5G)
and streams the data to a cloud-hosted Flask server for centralized analysis."

ONE PARAGRAPH:
Network Cell Analyzer is a two-part distributed system consisting of a native
Android application (written in Kotlin) and a Python Flask backend hosted
24/7 on Render cloud. The Android app reads cellular network information
directly from the device's TelephonyManager API every 10 seconds —
capturing operator, signal strength, network type (2G/3G/4G/5G), SINR,
frequency band, and cell ID — and transmits each measurement to the cloud
server over HTTPS. The server persists all measurements in a SQLite database,
tracks connected devices in real time with IP and MAC addresses, and exposes
both a REST API for statistics queries and a live-updating web dashboard.
Users can select any date range in the mobile app and fetch comprehensive
statistics including average connectivity time per operator and network
type, average signal power per device and network type, and SINR averages.

SLIDE 12 — CHALLENGES & HOW WE SOLVED THEM
--------------------------------------------------------------------------------
Title: "Challenges Overcome"

Three pain points with resolutions:

❗ Android 10+ blocks direct MAC address access
   ✓ Implemented NetworkInterface fallback + deterministic pseudo-MAC
     derived from Android ID — each device still has a consistent,
     unique MAC-formatted identifier.

❗ Phone and server on different networks
   ✓ Deployed server to Render cloud with HTTPS — works from any
     network (Wi-Fi, mobile data, anywhere).

❗ Render free tier cold-start delay (~30s after idle)
   ✓ Bumped OkHttp client timeouts from 5s to 45s so the first
     request after a sleep still succeeds.
